@PURPOSE: Provide special-purpose application-level containers.

@AUTHOR: Rohan Bhindwale (rbhindwa)

@DESCRIPTION: The 'baecs' package contains a small number of special-purpose
 application-level containers.  Of particular note are the 'baecs_journal' and
 'baecs_mappingmanager' components that provide a file-backed persistent
 journal, and an in-process manager of mmap pages, respectively.  Note that the
 'baecs_journalheader' and 'baecs_journalpageheader' components are intended
 for use internally by 'baecs_journal', and should not be used directly by
 client code.

/Hierarchical Synopsis
/---------------------
 The 'baecs' package currently has 5 components having 3 levels of physical
 dependency.  The list below shows the hierarchical ordering of the components.
 The order of components within each level is not architecturally significant,
 just alphabetical.
..
  3. baecs_journal

  2. baecs_journalheader

  1. baecs_journalpageheader
     baecs_journalparameters
     baecs_mappingmanager
..

/Component Synopsis
/------------------
: 'baecs_journal':
:      Provide a file-backed persistent journal.
:
: 'baecs_journalheader':
:      Provide a container for the header information of a journal.
:
: 'baecs_journalpageheader':
:      Provide a container for the header information of a journal page.
:
: 'baecs_journalparameters':
:      Provide a container for the file parameters of a journal.
:
: 'baecs_mappingmanager':
:      Provide an in-process manager of mmap pages
